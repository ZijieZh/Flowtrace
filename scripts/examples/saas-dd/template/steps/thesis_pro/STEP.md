# Thesis — Pro (Steelman)

Source skill: **cross-eval** (the steelman pass — "you must find the 3 strongest reasons to
approve"; the GO / PAUSE / STOP recommendation logic; disagreement is signal, not noise).

The first of the two-sided thesis, and a **fan-in from all five lenses**. This node — and its
contra twin — is built **before** valuation, on purpose: the cross-eval discipline is to argue
the case on the facts so the **price cannot anchor the argument**.

Build the **strongest case TO DO the deal**: buy beats build, the AI team is the prize, NRR and
margin are real, the cross-sell is landable, the seller is motivated (ask < last round). Record
the **key assumptions** the YES rests on and a **vote (APPROVE, Medium)**. Note where the lenses
*agree* — they all like the asset. Write `thesis_pro.json`.
